# Daily News Agent - 系統核心運行準則與需求記憶

每次執行任何修改或排程任務前，請務必回顧本文件以確保產出符合柔柔的原始需求。

## 核心四大板塊需求
1. **【國內頭條】(Headline News)** 
   - 定義核心：由媒體與編輯台判斷，當天社會影響力最大、最重要的國內外焦點大事。
   - 限制：具有權威性與重大性。**絕對不要**放娛樂八卦或即時小新聞，並且**僅限當天**的新聞。若當天真的沒有符合此等級的核彈大新聞，此區塊嚴格留空（不放），寧可空著也絕對不能拿舊聞或小新聞充數。
   - 去重：不可與前幾日的新聞重複。
2. **【國際頭條】(Headline News)**
   - 定義核心：由媒體與編輯台判斷，當天國際上影響力最大的焦點大事。
   - 限制：同國內頭條。**絕對不要**放即時小新聞，並且**僅限當天**的新聞。若沒有符合標準的國際大事，此區塊嚴格留空（不放）。
   - 去重：不可與前幾日的新聞重複。
3. **【熱門新聞】(Trending News)**
   - 定義核心：由讀者流量決定。點閱率、搜尋量或分享數最高的內容，反映大眾即時興趣（往往包含娛樂、八卦、突發或奇聞）。
   - 數量：5~10則。
   - 限制：必須是**當天**最高點閱的話題。
   - 去重：不可與前幾日的新聞重複。
4. **【台中天氣預報】與全台查詢**
   - 每日推播包含：台中每日攝氏溫度(度C)、降雨量、紫外線程度、一週溫度變化趨勢。
   - 前端互動：具備「即時查找全台各區域天氣功能」（包含溫度、降雨、紫外線）。

## 系統全域行為規範
- **防重複機制 (Deduplication)**：一定要紀錄歷史抓取連結或標題，每次產出報告前須比對並排除已發布過的新聞。
- **網頁同步 (Web Sync)**：報告內的連結必須能直接對應到儀表板或擷取的內容，讓使用者點進網址直接能看到該則報導。
- **定時發送排程 (Schedule)**：每日報告必須在 **台北時間 早上 8:00** 自動生成並發送至 LINE 與 Telegram。
